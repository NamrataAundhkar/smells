Unexploited Encapsulation

This smell arises when client DESIGN uses explicit type checks (using chained if-else or switch statements that check for the type of the object) instead of exploiting the variation in types already encapsulated within a hierarchy.

This smell is also known as "Simulated polymorphism" [SISSy], "Improper use of switch statement" [Fowler], and "instanceof checks" [Van].

Reference

Girish Suryanarayana, Ganesh Samarthyam, and Tushar Sharma, Refactoring for Software Design Smells: Managing Technical Debt, 1st ed. Morgan Kaufmann, 2014.
