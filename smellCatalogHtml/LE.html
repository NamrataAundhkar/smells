Leaky Encapsulation

This smell arises when an abstraction "exposes" or "leaks" implementation details through its public interface.

It is also referred as "Leaking implementation details in API" [Bloch].

Reference

Girish Suryanarayana, Ganesh Samarthyam, and Tushar Sharma, Refactoring for Software Design Smells: Managing Technical Debt, 1st ed. Morgan Kaufmann, 2014.
